# LLMで回す社内IT英語（ワークフロー）

LLMは「会話相手」よりも、**文章作成・添削・言い換え・不足情報の洗い出し**で最大効率が出ます。

> 重要：社外秘/個人情報/顧客情報/生ログ/URL/チケット番号などは、そのまま入れずに伏せ字や一般化で。

---

## 毎日20分（最小）

初週は [getting-started.md](getting-started.md) の Day 1–7 に従うと迷わない。

- 7分：フレーズ置換（`docs/phrasebook.md`から5個選ぶ→主語/期限/対象を入れ替える）
- 7分：チケット用短文（3〜5文）を書く→LLM添削
- 6分：英語ドキュメント1段落を要約（自分→LLMでチェック）

## 週2〜3回（実戦）

- 「サポートチケット」か「障害/変更連絡」を1本作って添削
- よく間違える表現を10個だけメモして翌週も反復

---

## コピペ用プロンプト集

### 1) チケット文章を作る（不足情報を質問させる）

日本語を貼る前に、まずは情報を伏せる：
- 会社名/顧客名/個人名 → `[COMPANY]` `[CUSTOMER]` `[PERSON]`
- URL/ドメイン/IP → `[URL]` `[IP]`
- チケット番号 → `[TICKET]`
- ログ → 代表的な1〜2行だけにする（または要約）

プロンプト：

```
以下の状況を英語のサポートチケットにして。
構成は Summary / Environment / Steps to reproduce / Expected / Actual / Impact / Troubleshooting tried / Logs。
不足情報があれば、最初に質問してから作成して。
日本語：
...
```

### 2) 添削（学習になる形）

```
この英文を、(1)文法ミス修正 (2)より自然 (3)より丁寧 の3案で直して。
変更点を箇条書きで理由つき。
原文：
...
```

### 3) 会議ロールプレイ（ミスを固定化させない）

```
あなたは海外チームのメンバー。私は社内IT。
会話は英語で、進捗確認→課題→依頼→次アクションの順で5往復。
私の英語が不自然なら「会話を止めずに」自然に言い直して続けて。
最後に改善点3つと必修フレーズ5つ。
```

### 4) ドキュメント読解（要点だけ抜く）

```
以下の英語ドキュメントを、社内IT向けに日本語で要約して。
- 手順（箇条書き）
- 注意点/前提条件
- 失敗しがちなポイント
- 用語ミニ辞書（5語）
英文：
...
```

## ありがちな落とし穴（回避策）

- 生成文をそのまま最終稿にする → 重要事項は必ず人間が確認（数字、条件、影響範囲）
- 新しい表現を増やしすぎる → 10個だけ反復（量より定着）
- 丁寧さが過剰/不足 → 「丁寧/標準/カジュアル」の3案を出させて選ぶ
